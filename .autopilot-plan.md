## Phase 2: API Routes

### 2a. Product & Category APIs
- `GET /api/coffee/products` - List products (public, with category filter, search)
- `GET /api/coffee/categories` - List categories (public)
- `GET/POST/PATCH/DELETE /api/admin/coffee/products` - Admin CRUD
- `POST /api/admin/coffee/categories` - Admin category CRUD

### 2b. Cart APIs
- `GET /api/coffee/cart` - Get user's cart (requires auth + coffee_access)
- `POST /api/coffee/cart` - Add/update item (requires auth + coffee_access)
- `DELETE /api/coffee/cart` - Remove item or clear cart (requires auth + coffee_access)

### 2c. Order APIs
- `POST /api/coffee/orders` - Submit order from cart (requires auth + coffee_access)
- `GET /api/coffee/orders` - List user's orders (requires auth + coffee_access)
- `GET /api/coffee/orders/[id]` - Order detail (requires auth, owner check)
- `POST /api/coffee/orders/[id]/reorder` - Reorder (add items back to cart)

### 2d. Application API
- `POST /api/coffee/apply` - Submit coffee application (requires auth)

### 2e. Admin APIs
- `GET/PATCH /api/admin/coffee/applications` - List & approve/reject applications
- `GET/PATCH /api/admin/coffee/orders` - List & update order status

### 2f. Favorites API
- `GET/POST/DELETE /api/coffee/favorites` - Toggle favorites (requires coffee_access)
